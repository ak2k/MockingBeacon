# NCS 3.2.4 migration research appendix

Reference material for the migration plan. Consult on-demand — NOT reading material. Agents should only read this when hitting an unexpected issue or needing to understand the reasoning behind a plan decision.

## Research summaries

Condensed findings from 13 research streams run during plan development. Each stream produced 500-2500 words; summaries below preserve only the load-bearing facts.

**WIP branch review (c46c3d7):** The prior straight-jump attempt produced 5 lines of correct semantic work (`BT_LE_ADV_OPT_CONN` rename in zephyr_hardware.cpp + gatt_glue.c, `bt_hci_cmd_alloc` swap, west.yml bump, narrow chmod hook) buried under 267 lines of drive-by clang-format reflow in gatt_glue.c. Stalled at `CONFIG_BT_BUF_EVT_RX_COUNT` BUILD_ASSERT without touching: `.recycled` callback, RAM audit, Kconfig deprecations, ZMS explicit enables, or the narrow chmod scope that would have broken DFU builds.

**NCS 2.9→3.2 migration research:** The jump crosses three Zephyr majors (3.7→4.0→4.1→4.2) and three NCS majors. Zephyr 4.0 is the largest single-release diff — advertising resumption removed, `BT_LE_ADV_OPT_CONN` introduced. Zephyr 4.1 auto-computes `BT_BUF_ACL_RX_COUNT` (= `BT_MAX_CONN + 1`). Zephyr 4.2 replaces `bt_hci_cmd_create` with `bt_hci_cmd_alloc`. NCS 3.0 requires sysbuild (multi-image/child-image removed). NCS 3.2 auto-assigns MCUboot image IDs. Nordic + Zephyr docs + Argenox all recommend sequential per-release migration; no "just jump" path.

**Beacon samples divergence review:** Our `bt_hci_cmd_alloc` pattern matches `samples/bluetooth/hci_pwr_ctrl/src/main.c:82-111` exactly — idiomatic. `bt_ctlr_set_public_addr` unchanged. `BT_GATT_SERVICE_DEFINE` still canonical. Our dynamic SMP registration is ahead of upstream sample posture. Key divergences worth folding in: `.recycled` callback + workqueue adv restart (per `samples/bluetooth/peripheral_lbs/src/main.c:53-67`); `BT_LE_ADV_CONN_FAST_1` macro for settings-mode intervals; `bt_hci_err_to_str()` log improvements (deferred to TODO); `BUILD_ASSERT(IS_ENABLED(CONFIG_BT_HAS_HCI_VS))` (deferred). Defer post-migration: `bt_le_ext_adv_*` API for simultaneous adv sets.

**performance-oracle (RAM/flash):** Phase 1g RAM audit must be measurement-driven, not checklist-driven. Order: `.config` diff → `nm --size-sort` diff → `size -A` per subsystem, THEN apply levers. Bss budget ≤ 23 KB hard / 22 KB target (not 22 KB hard — Zephyr 4.0 alone expected to add ~1.5-2.5 KB). Text ≤ +3% target / +5% hard (not +5%/+10% — 7 KB regression on 192 KB nrf52810 would be catastrophic). Real RAM-saving levers: `CONFIG_LOG_BUFFER_SIZE=512`, `CONFIG_CBPRINTF_REDUCED_INTEGRAL=y`, `CONFIG_BT_HCI_ACL_FLOW_CONTROL=n`, `CONFIG_BT_LONG_WQ=n`, `CONFIG_BT_CTLR_DUP_FILTER_LEN=0`. Drop from audit list: `MBEDTLS_PSA_KEY_SLOT_COUNT` (no PSA linked without SMP), `BT_CTLR_DATA_LENGTH_MAX` (already DLE=n), `BT_BUF_ACL_TX_SIZE` (already 27 not 251). Workqueue: dedicated K_THREAD_STACK 768 B; fallback to `SYSTEM_WORKQUEUE_STACK_SIZE=1280` if tight.

**security-sentinel:** Pre-existing issue: `sysbuild/mcuboot.conf` enables ECDSA-P256 but doesn't set `CONFIG_BOOT_SIGNATURE_KEY_FILE` → MCUboot falls back to public upstream dev key; anyone can sign DFU images. Decision: keep dev key for prebuilds, track in TODO.md. Critical migration-time concern: `.recycled` timing race could leak `authorizedGatt=1` from client A to client B if recycled fires before disconnected clears the flag — MUST clear in both callbacks. Other findings: audit `bt_gatt_authorization_cb` struct layout (new fields = zero-init = silent authorize), hash-diff 5 critical west2nix pins (hal_nordic, nrfxlib, mbedtls, trusted-firmware-m, mcuboot), `strings | grep -iE 'auth|key|secret'` diff against baseline to catch accidental log leaks.

**architecture-strategist:** Original plan's Phase 1 over-bundled 6 concerns behind one gate. Split into 1α (compile-green: lockfile/chmod/Kconfig/ZMS/sysbuild) + 1β (behavior-green: advertising/RAM audit/tests). Original plan's single-long-lived-branch was wrong — per-phase merges to main are correct (main has weekly release cadence via ncs-upgrade workflow; 5-day freeze unacceptable). **Hidden architectural concern**: `IHardware::start_*_adv()` is synchronous today; naive `.recycled` + workqueue implementation silently makes it async, breaking StateMachine assumptions. Fix: semaphore barrier inside zephyr_hardware.cpp preserves sync contract. Also: per-commit bisect granularity is requirement not preference (promoted). MCUboot image ID risk re-rated MEDIUM/Critical (was LOW/Critical) — detectability is low even if likelihood is low.

**code-simplicity-reviewer:** Plan v1 had padding. Cuts: 3 of 6 opportunistic fold-ins (`bt_hci_err_to_str`, `BUILD_ASSERT(HCI_VS)`, comment cleanup — move to TODO); 3 risk-table padding rows (lockfile drift, clang-format, DFU/non-DFU); 5 soft acceptance criteria (build time, kconfig warnings, format idempotent, CI workflow green, soft flash target); Phase 2 rollback ceremony (Phase 2 is housekeeping — no rollback trigger). Load-bearing (DO NOT CUT): `.recycled` + workqueue, nrf52810 RAM audit, MCUboot image ID grep, baseline capture, 5 integration test scenarios.

**data-integrity-guardian:** Partitions defined statically in board DTS (not pm_static.yml). Real integrity risk: NCS 3.0+ sysbuild-PM integration can silently synthesize `pm.yml` overlay that overrides DTS offsets — scan `find . -name 'pm.yml' -o -name 'pm_static.yml'` post-migration; halt on any hit. `BT_SETTINGS=n` — `CONFIG_SETTINGS_ZMS_MAX_COLLISIONS_BITS` is N/A; we use direct `zms_read/write` with hand-assigned uint16_t IDs. **Real bug found:** `tests/zms_persist/src/main.c` ID map doesn't match production (`ID_TIME=0x0B` in test vs. `ID_status_NVS=0x0B` in production). Fix: extract NVS ID enum to shared header `src/beacon_nvs_ids.hpp`; bsim test must iterate every production ID. nrf52810 DFU not applicable (56 KB slots < 150 KB image). `zms_calc_free_space()` telemetry is nice-to-have.

**deployment-verification-agent:** Produced pre-merge checklist, post-merge verification commands (fresh clone), rollback SOP (`git revert -m 1`), monitoring plan (7-day signals: CI wall-clock, Cachix hit ratio, Kconfig warnings), developer communication draft (action-required list for dev pulls). All folded into plan's Deployment Checklist section.

**julik-races-reviewer (frontend-races expertise applied to workqueue/callback races):** Plan's naive hardcoded `BT_LE_ADV_CONN_FAST_1` in `adv_work_handler` is wrong for this multi-mode firmware (AirTag/FMDN/iBeacon are non-connectable with different intervals). Required: (1) multi-mode dispatch in handler based on current/pending adv mode; (2) `k_work_cancel_sync(&adv_work)` before every state-machine mode transition (`beacon_state.cpp:213, 260, 301, 356, 378`); (3) `adv_allowed()` predicate guarding against ShuttingDown/FirmwareUpload states; (4) bounded retry on `-EAGAIN`/`-ENOMEM` (max 5 attempts, 100ms backoff — never infinite); (5) single `k_work` struct + mode dispatch, not multiple structs; (6) document that `.recycled` is restart path, not start path — initial adv still comes from `StateMachine::initialize()`.

**spec-flow-analyzer:** Caught 7 flow gaps: (1) developer onboarding post-merge needs explicit `west update` runbook; (2) CI failure on no-hardware boards needs tier distinction; (3) mid-migration regroup protocol undefined; (4) integration test triggers unwired (no `tests/ble_client/test_rapid_reconnect.py` exists); (5) CI workflow grep needs specific file:line targets (release.yml:171 awk parse, 186-189 DFU slot sizes); (6) post-merge local env needs cache warmup + stale-pycache warning; (7) **confirmed stale memory**: `nix build .#firmware-nrf52832-dfu` now works on main (commit 42793bf on 2026-04-15 fixed the `zephyr_module.py TypeError`). All folded in.

**framework-docs-researcher:** Answered 8 specific Zephyr 4.2 API questions against NCS v3.2.4 source at `/Users/akirby/Downloads/zephyr`. Key facts: `.recycled` timing strictly after `.disconnected` post conn release, treat as restricted context (use `k_work_submit`). `BT_BUF_EVT_RX_COUNT > BT_BUF_ACL_TX_COUNT` BUILD_ASSERT at `include/zephyr/bluetooth/buf.h:168-172`. MCUboot image IDs surfaced via `CONFIG_MCUBOOT_APPLICATION_IMAGE_NUMBER`; sysbuild log line `"Sysbuild assigned MCUboot image IDs"` greppable in build output. `MBEDTLS_PSA_KEY_SLOT_COUNT` default was 32 (implicit), became 16 (4.1+), set to 4 for ~1 KB savings IF PSA is linked. `ZMS_LOOKUP_CACHE`: 8 B RAM per entry; size 16 = 128 B for nrf52810. `BT_LE_ADV_CONN_FAST_1`=30-60 ms, `_FAST_2`=100-150 ms. `_bt_gatt_ccc` → `bt_gatt_ccc_managed_user_data` rename is TRANSPARENT for `BT_GATT_CCC(...)` macro consumers (our case) — no action needed.

**best-practices-researcher:** Mined 2025-2026 community reports for gotchas not in official docs. NCS 3.3.0-rc2 exists but not yet GA as of 2026-04-16; 3.2.4 is terminal 3.2.x patch. Key findings beyond official docs: device-tree partition drift on NCS 3.0→3.2 for custom boards (DevZone 126561 — our custom boards must be diffed against upstream base); `bt_enable -EAGAIN` sporadically after MCUmgr reset on 3.2.0 (DevZone 105423, root cause BT buffer sizing with WDT timing); nRF54L15 needs `CONFIG_BOOT_MAX_IMG_SECTORS=256` or MCUboot partition sizing fails (DevZone 126140); spurious `"No valid legacy adv to resume"` log in Zephyr 4.2 under BT_EXT_ADV (issue #94954, cosmetic); `nrfjprog` → `nrfutil device` hard break at NCS 3.0 (not 3.2) — verify flake dev shell exposes nrfutil ≥2.8.8. BabbleSim + Bumble compat unchanged.

## Decision reasoning

Decisions made during plan development with full reasoning chain. Plan has the what; this has the why.

| Decision | Alternatives considered | Chosen | Why |
|----------|-------------------------|--------|-----|
| Migration strategy | Straight-jump vs. incremental 3.0→3.1→3.2 | Incremental | WIP already demonstrated straight-jump failure mode (one error surfaces, rest hidden). Three independent sources recommend incremental. Per-stage wall-clock cost is ~20 min vs. multi-day debugging benefit. |
| Branch strategy | Single long-lived branch + final PR vs. per-phase merges | Per-phase merges | `main` has weekly release cadence (ncs-upgrade workflow). 5-day freeze blocks releases. Intermediate states (3.0.2, 3.1.x) are valid not broken. Merged history gives real bisect points. Contradicted simplicity-reviewer's "reviewers find staged hard" claim — but simplicity-reviewer's claim was wrong (intermediate states are not broken-by-design). |
| IHardware contract | (a) sync-via-semaphore, (b) explicit async with polling, (c) document and audit callers | (a) | Preserves existing StateMachine callers (~8 sites across mode transitions). Cost: ~100 B for semaphore + one context switch per call. Option (b) requires touching every caller; option (c) is error-prone (no compiler help). |
| Workqueue stack | Dedicated K_THREAD_STACK 768 B vs. bump `SYSTEM_WORKQUEUE_STACK_SIZE=1280` | Dedicated (primary), bump fallback | System WQ already used by BT host notifications + ZMS. `bt_le_adv_start` can consume 400-600 B stack. Dedicated avoids overflow class; SYSTEM_WQ bump is 256 B cheaper if RAM-tight. |
| ZMS backend compat | Keep `CONFIG_SETTINGS_ZMS_LEGACY=y` (compat) vs. fresh backend | Fresh backend | User confirmed no deployed devices → greenfield. Legacy compat unnecessary. Cheaper to drop. |
| Dev-key MCUboot signing | Fix now (option a) vs. defer (option b) | Defer | User decision 2026-04-16: acceptable for prebuilds; track in TODO.md. Migration does not change signing config; signature-isolation test SKIPPED. |
| Hardware tests | Do with hardware vs. Bumble+bsim vs. halt migration | Bumble+bsim+build-audits | User confirmed no hardware available. `.recycled` wiring fully testable via Bumble virtual (real BT host stack, real conn lifecycle). DFU swap at runtime NOT fully simulable — gap tracked in TODO.md. Halting blocks all other progress indefinitely. |
| Fold-ins from beacon samples | Fold all 6 vs. split required/opportunistic | Split — fold 3, defer 3 | Required (coupled to Zephyr 4.0): `.recycled` + workqueue, `BT_LE_ADV_CONN_FAST_1`. Opportunistic in touched file: dead-Kconfig cleanup. Move to TODO: `bt_hci_err_to_str`, HCI_VS BUILD_ASSERT, comment cleanup. Per simplicity-reviewer: opportunistic fold-ins = scope creep unless coupled. |
| Phase 1 split | Single-phase vs. 1α (compile) + 1β (behavior) | Split | Bundling compile-system work with advertising behavior work defeats bisect. Split gives clean failure-class attribution: 1α failure = build system, 1β failure = behavior. |
| adv_work_handler dispatch | Hardcoded `BT_LE_ADV_CONN_FAST_1` vs. mode-aware | Mode-aware | Firmware has 4 adv modes (Settings/AirTag/FMDN/iBeacon) with different params. Hardcoded = stale-mode advertising after any state transition. Mode dispatch reads `pending_adv_mode_` and applies per-mode params. |
| Bounded retry semantics | Infinite vs. bounded (5 attempts × 100ms) | Bounded | Infinite retry on `-EAGAIN`/`-ENOMEM` = stack thrash + battery drain + silent hang. Bounded gives escalation signal. |
| `authorizedGatt` clearing | `.disconnected` only (plan v1) vs. both callbacks | Both | Race window: `.recycled` fires before `.disconnected` completes cleanup. Without belt-and-suspenders, new client inherits prior auth. Defense-in-depth is ~3 lines. |
| Test authorship | Central test-writer agent vs. implementor writes own tests | Implementor writes, reviewer validates via test-injection | Tests written by someone who wasn't there when the bug was found tend to test the wrong thing. Implementor knows the bug shape. Reviewer injects fault to confirm test fails as expected. |
| Reviewer veto scope | Block anything red vs. block plan violations + acceptance failures | Plan violations + acceptance failures | "Anything red" including deprecation warnings = too many human unblocks. Plan + acceptance gives clear contractual authority without noise. |
| Worktree isolation | One per agent vs. one per phase vs. shared checkout | One per phase | One-per-agent = disk waste + merge overhead. Shared = conflicts. Per-phase balances: within-phase sub-tasks have clear file-ownership boundaries. |
| Coordinator context | Continue this conversation vs. fresh `/ce:work` session | Fresh session | Tests the plan as a spec. If plan is solid, fresh coordinator executes it. Implementors/reviewers are always fresh-context by default; only coordinator question. Fresh = resumable + cleaner. |

## Symptom → likely cause map

For agents hitting unexpected issues mid-migration. Format: error/symptom you see → most-likely-cause → quick diagnostic.

| Symptom | Likely cause | Diagnostic / fix |
|---------|--------------|------------------|
| `BUILD_ASSERT: Increase Event RX buffer count...` (zephyr 4.x) | `BT_BUF_EVT_RX_COUNT ≤ BT_BUF_ACL_TX_COUNT` | Set `CONFIG_BT_BUF_EVT_RX_COUNT=4` (or one more than ACL_TX); BUILD_ASSERT is at `include/zephyr/bluetooth/buf.h:171`. |
| BLE advertising works for first connection only then silent | Missing `.recycled` callback | Zephyr 4.0 removed auto-resume. Add `.recycled = recycled_cb` to `bt_conn_cb`; handler must submit work to queue, not call `bt_le_adv_start` inline. |
| `bt_le_adv_start` returns `-ENOMEM` from `.disconnected` | Calling from wrong callback | Conn object still held in `.disconnected`. Move to `.recycled` (fires after conn release). |
| Advertising runs wrong mode after key rotation | Stale work item in queue | Missing `k_work_cancel_sync(&adv_work)` before state transition. Cancel before `adv_stop`/`bt_disable` in every transition site (beacon_state.cpp:213, 260, 301, 356, 378). |
| New client appears pre-authorized | `authorizedGatt` not cleared in `.recycled` | Add `authorizedGatt = 0` to `.recycled` callback (not just `.disconnected`). |
| `zephyr_module.py TypeError: NoneType += str` during Nix build | Missing git init of manifest dir | NOT a migration bug — already fixed on main by commit 42793bf. If seeing this post-Phase-1b, the chmod widening regressed the `git init` block at `flake.nix:107-111`. |
| `find_program: imgtool.py not found` during DFU build | chmod hook too narrow | Phase 1b widened to cover `bootloader/mcuboot/scripts`. If seeing this, the widening didn't take. |
| `nix build .#firmware-nrf52832-dfu` crashes on swap | MCUboot image ID mismatch | DevZone 127216 class. Grep `src/` + `boards/` for hardcoded image IDs; replace with `CONFIG_MCUBOOT_APPLICATION_IMAGE_NUMBER`. Check sysbuild log for "Sysbuild assigned MCUboot image IDs". |
| `bt_enable` returns `-EAGAIN` post-DFU reset | BT buffer + WDT timing | DevZone 105423. Drop `BT_BUF_ACL_TX_SIZE` and `BT_CTLR_DATA_LENGTH_MAX` together (never asymmetric) from 251 to 179. |
| nrf52810 bss > 23 KB | Zephyr 4.x default inflation | `.config` diff vs. baseline: look for new default increases in `BT_*`, `LOG_*`, `NET_BUF_POOL*`. Apply measurement-first audit (Phase 1g). Don't guess. |
| nrf54l15dk DFU "image too large" | `BOOT_MAX_IMG_SECTORS` too small | DevZone 126140. Set `CONFIG_BOOT_MAX_IMG_SECTORS=256` for nRF54L15 DFU variant. |
| Log: `"No valid legacy adv to resume"` | Cosmetic bug in Zephyr 4.2 | GitHub issue #94954. Don't chase — fixes in future 4.2.x. |
| Test `tests/zms_persist/` passes on old IDs but fails on new | ID enum drift between test and production | Extract to `src/beacon_nvs_ids.hpp`, include from both sides. Test must iterate all production IDs. |
| `pm.yml` appears in `build*/` unexpectedly | NCS 3.2 sysbuild-PM auto-synthesis | Audit: `find . -name 'pm_static.yml' -o -name 'pm.yml'`. Halt if hits; may silently override DTS partition offsets. |
| `CONFIG_BT_SETTINGS_CCC_LAZY_LOADING` unknown | Deprecated Kconfig | No-op when `BT_SETTINGS=n`. Delete from prj.conf. |
| `CONFIG_BT_DEBUG_NONE` warning | Deprecated Kconfig | Replace with `CONFIG_BT_LOG_LEVEL_OFF=y`. |
| `nrfutil device` not found | Pre-3.0 tooling | NCS 3.0+ requires `nrfutil device ≥ 2.8.8`. Check `flake.nix` devShell. |
| `CONFIG_BT_BUF_ACL_RX_COUNT` deprecated warning | Zephyr 4.1 auto-compute | Remove explicit value; now computed as `BT_MAX_CONN + 1`. Use `CONFIG_BT_BUF_ACL_RX_COUNT_EXTRA` only if extras needed. |
| `struct _bt_gatt_ccc` warning | Zephyr 4.2 rename | If using `BT_GATT_CCC(...)` macro: no action (transparent). If using raw struct: rename to `bt_gatt_ccc_managed_user_data`. |
| Test hangs on connection | Workqueue stack overflow | THREAD_ANALYZER check. `bt_le_adv_start` path can take 600 B stack. Either use dedicated K_THREAD_STACK 768 B or bump `SYSTEM_WORKQUEUE_STACK_SIZE=1280`. |
| Unexpected boards' text size drifts >1% | Kconfig default flipped silently | `.config` diff vs. baseline. Explain or revert. Architecture-strategist's "negative acceptance criteria" — unchanged boards should not drift. |

## Things considered-and-rejected (save re-research time)

- **Extended advertising API migration** (`bt_le_ext_adv_*`) during migration — rejected, tracked in TODO.md. Medium-effort cross-cutting refactor, legacy `bt_le_adv_start` still works in 3.2.x.
- **Per-attribute `BT_GATT_PERM_*`** with SMP pairing instead of `bt_gatt_authorization_cb` — rejected, deliberate application-layer auth design; revisit only if adding pairing.
- **Pause for NCS 3.3** — rejected, 3.3 adds Zephyr 4.4 ZMS→kvss header move. Waiting compounds breaking-change debt.
- **Salvage WIP branch** — rejected, 5 lines of good work buried under 267 lines of drive-by reflow. Reproducing is cheaper than untangling.
- **Extend bsim_dfu to full MCUboot simulation** (native_sim + CONFIG_FLASH_SIMULATOR) — rejected, 0.5-1 day engineering spend for uncertain coverage. Hardware DFU swap validation tracked in TODO.md instead.
- **Signature-isolation test enabled now** — rejected, requires setting `CONFIG_BOOT_SIGNATURE_KEY_FILE` which is a separate decision (see TODO.md). Skip during migration; re-enable with signing-key work.
- **`MBEDTLS_PSA_KEY_SLOT_COUNT` as a RAM lever** — rejected, PSA crypto is not linked on our non-DFU path (no SMP → no PSA consumers). Verify via `.config` diff; only consider in DFU variant.
